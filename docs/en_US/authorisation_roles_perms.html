Permission Management

Since release 1.5 NagVis contains a very detailed permission management. It is a role based concept where Users can be assigned to one or several roles while the roles have permissions assigned.

The permissions of the different roles of a user are summarized and controll the actions which the user is able to perform.

Roles

Default Roles

On a clean installation there are four roles defined:

  • Administrators: Full permissions on the whole installation
  • Managers: Read and write access to all maps, automaps and rotations. Allowed to create new maps, automaps and rotations.
  • Users (read-only): Read access to all maps, automaps and rotations in the installation
  • Guests: Read access to all demo maps, automaps and rotations

These default roles should be ok for the beginning. You may delete these users and roles but please be aware not to delete the last administrative user and role.

Customizing the Roles

It is possible to create new roles for customizing the permissions of users and collection of users.

For example it is easy to create a role for all SAP administrators which then can only access the SAP related maps.

Permissions

The permissions are hardcoded in NagVis. One permission is assembled of three things:

  • Module: The related module (Map, Automap, Rotation, ...)
  • Action: The related action (view, edit, delete, ...)
  • Object: The related object (A particular user, map, ...)

Each of those three things can be set by a dedicated name or alternatively by a wildcard "*" which then matches all modules, actions or objects.
