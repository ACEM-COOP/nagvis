backend~Waehle ein Backend. z. Zt. sind die Backends "NDOMY" (schnell und stabil, zieht die Daten aus einer vorhandenen Nagios NDO MySQL Datenbank) und HTML (langsam und eher unzuverlässig, parsed die Nagios CGIs) verfügbar.
language~Waehle deine Sprache.
defaultIcons~Standard Iconset.
rotateMaps~Rotiere Maps automatisch.
maps~Zu rotierende Maps. Beispiel: server,lan,wan
displayHeader~Header anzeigen.
headerCount~Anzahl der Links pro Zeile im Header.
autoUpdateFreq~WUI legt beim bearbeiten regelmäßig ein Backup der Datei an bzw. speichert automatisch ab. Der Intervall kann hier eingestellt werden.
checkConfig~If set to 'Yes', forces a check of the current configuration. MUST be done after any change.
useGDLibs~GDLib benutzen? GD wird nur zum zeichnen von "Line" typen benötigt, wenn Probleme mit GD auftreten kann GD hier mit "0" deaktiviert werden, alle anderen Typen funktionieren auch ohne GD. Lines stehen dann aber nicht mehr zur Verfügung.
refreshTime~Map refresh time (in Sekunden).
debug~Debug an oder aus?
debugStates~Debug Status?
debugCheckState~Debug Statuscheck?
debugFixIcon~Debug FixIcon?
base~Voller lokaler Pfad zum NagVis Basis Verzeichnis.
cfg~Lokaler Pfad zum NagVis etc Verzeichnis.
icon~Lokaler Pfad zum NagVis iconsets Verzeichnis.
map~Lokaler Pfad zu den Hintergrund Bildern (*.png)
mapCfg~Lokaler Pfad zu den Map Config Dateien (*.cfg)
htmlBase~HTTP Pfad zu NagVis (/nagios/nagvis).
htmlCgi~HTTP Pfad zu den Nagios CGIs (/nagios/cgi-bin).
htmlIcon~HTTP Pfad zum NagVis iconsets Verzeichnis.
htmlMap~HTTP Pfad zu den NagVis Maps (/nagios/nagvis/maps/).
htmldoku~HTTP Pfad zur NagVis Doku.
header~Name der Header include Datei.
index~Include Datei fuer die Standard Seite.
dbHost~NDO Backend: Host der NDO Datenbank (z.B. localhost).
dbPort~NDO Backend: Port der NDO Datenbank (default 3306).
dbName~NDO Backend: Name der NDO Datenbank.
dbUser~NDO Backend: User für den NDO Datenbankzugriff, der User braucht nur read (SELECT) Rechte!.
dbPass~NDO Backend: Passwort des Users für den Datenbankzugriff.
dbPrefix~NDO Backend: Kürzel vor den Tabellennamen in der NDO DB (z.B. "ndo_")
dbInstanceId~NDO Backend: Instanz-ID der NDO installation (wird nur bei mehreren NDO installationen benötigt, normalerweise "1")
cgiUser~HTML Backend: Der User mit dem NagVis die CGIs aufruft (sollte 'allowed for all services', 'allowed for all hosts' sein)
cgi~HTML Backend: Voller lokaler Pfad zu den Nagios CGIs (/usr/local/nagios/sbin).
version~NagVis Version
title~Titel der NagVis Seiten im Browser
